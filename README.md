# 상담 일지 작성 도구

상담사가 메모한 키워드로 표준화된 상담 일지 초안을 자동 작성하는 도구입니다.
상담 데이터는 서버에 저장하지 않고, 생성 즉시 화면 표시 및 .txt 다운로드로만 처리합니다.

---

## 1. 설치

```bash
pip install -r requirements.txt
```

## 2. 키 / 접근코드 설정

`.streamlit/secrets.toml.example` 파일을 `.streamlit/secrets.toml` 로 복사한 뒤 값을 채웁니다.

```toml
GEMINI_API_KEY = "발급받은-키"
ACCESS_CODE = "상담사들에게-공유할-접근코드"
```

> `secrets.toml` 은 `.gitignore` 에 등록되어 있어 깃에 올라가지 않습니다. 키가 새어나갈 걱정이 없습니다.

## 3. 실행

```bash
streamlit run app.py
```

---

## 📱 핸드폰에서 사용하기

화면은 모바일에서 위→아래로 자동 정렬되어 폰에서도 그대로 쓸 수 있습니다. 접속 방법은 두 가지입니다.

**방법 A — 같은 와이파이에서 바로 접속 (간단)**
1. PC에서 `streamlit run app.py` 실행
2. 터미널에 표시되는 `Network URL`(예: `http://192.168.0.x:8501`)을 폰 브라우저에 입력
3. PC와 폰이 같은 와이파이에 있어야 합니다.

**방법 B — 인터넷 어디서든 접속 (Streamlit Community Cloud, 무료)**
1. 코드를 GitHub 저장소에 올립니다. (`secrets.toml` 은 올리지 않습니다)
2. share.streamlit.io 에서 저장소를 연결해 배포합니다.
3. 배포 설정의 **Secrets** 칸에 `GEMINI_API_KEY` 와 `ACCESS_CODE` 를 입력합니다.
4. 생성된 주소를 폰 즐겨찾기에 추가하면 앱처럼 쓸 수 있습니다.

## 🔑 API 키 상황별 안내 (무료 vs 유료)

이 도구는 **관리자가 키 하나를 등록**하면 동료 상담사는 접근코드만으로 사용하는 구조입니다.
키를 발급할 때 본인 상황에 맞춰 아래를 참고하세요.

| 상황 | 데이터 처리 | 권장 여부 |
|---|---|---|
| **무료 티어** (결제 미연결) | 입력 내용이 Google 제품·모델 개선에 사용될 수 있고, 검토자가 볼 수 있음 | ⚠️ 민감한 상담에는 비권장 |
| **유료 티어** (결제 연결) | 프롬프트·응답을 모델 학습에 사용하지 않음(안전·법적 목적의 제한적 로그만 남음) | ✅ 권장 |

**유료 티어로 전환하는 법**
1. Google AI Studio → 해당 프로젝트에서 **결제(Billing) 연결**
2. Google Cloud Console에서 **월 지출 한도(spend cap)**를 걸어두면 예상치 못한 과금을 막을 수 있습니다.
3. 이 도구는 비용이 낮은 `gemini-2.5-flash` 를 사용하므로, 상담 일지 정도의 사용량은 매우 저렴합니다.

> 참고: 유럽(EEA)·스위스·영국에서는 무료 티어에도 유료 수준의 데이터 정책이 적용되지만, 한국은 해당하지 않으므로 위 표대로 유료 티어를 권장합니다.

**동료마다 각자 자기 키를 쓰게 하고 싶다면?**
현재는 중앙 키 방식입니다. 각자 키를 입력하는 방식(BYOK)으로 바꾸려면 로그인 화면에 키 입력란을 추가하면 되며, 필요 시 별도로 안내드릴 수 있습니다.

## 🛠 상담 유형 수정하기

상담 주제 목록은 `constants.py` 의 `COUNSELING_TOPICS` 한 곳에 모여 있습니다.
선생님들과 논의 후 이 리스트만 수정하면 화면에 바로 반영됩니다.

---

## 📂 파일 정렬 / 회차 관리

서버에 저장하지 않는 대신, **파일명을 정렬 친화적으로** 만들어 폴더 관리만으로 회차를 파악할 수 있게 했습니다.

```
A군_2026-06-25_1회차_143052.txt
A군_2026-07-02_2회차_091230.txt
B양_2026-06-25_1회차_153340.txt
```

- 이니셜이 맨 앞 → 같은 학생끼리 자동으로 모임
- 날짜가 그다음 → 학생 안에서 시간 오름차순 정렬
- 회차 번호는 입력칸에 직접 적으면 들어가고, 비우면 생략됨
- 끝의 시분초 덕분에 같은 날 두 번 상담해도 파일명이 겹치지 않음

PC/폰 파일 앱에서 **이름순 정렬**만 켜두면 위처럼 자동으로 줄섭니다.

## 🔗 이전 회차 이어가기 (연속성)

새 일지를 쓸 때, 입력 화면의 **'이전 회차 일지 첨부'**에 지난번 저장한 .txt 를 올리면
AI가 그 맥락을 참고해 이번 회기의 흐름을 이어 작성합니다.
파일은 이번 생성에만 쓰이고 서버에 저장되지 않습니다.

## 🗂 일지 양식 바꾸기 (JSON)

양식은 **'양식 만들기' 전용 페이지**에서 만듭니다. (앱 실행 후 왼쪽 사이드바 메뉴)

- 항목을 추가/삭제하고 위아래로 순서를 바꿀 수 있습니다.
- 각 항목에 **작성 가이드**(선택)를 달면 AI가 그 안내를 참고해 일지를 씁니다.
  (예: '추후 계획' 항목에 "다음 회기 목표와 일정 포함")
- 미리보기로 확인한 뒤 **'JSON 파일로 저장'**을 누르면 양식이 파일로 저장됩니다.
- JSON 문법은 전혀 몰라도 됩니다. 화면에서 만들면 JSON은 자동 생성됩니다.

저장한 양식은 **메인 화면 사이드바의 '양식 JSON 불러오기'**에 올리면 적용됩니다.
기관에 정해진 서식이 생기면 한 번 만들어 저장해두고, 이후엔 그 파일만 올려 재사용하세요.

양식 JSON 구조 (참고 — 직접 작성할 필요 없음):

```json
{
  "name": "우리기관 표준 상담일지",
  "sections": [
    {"title": "기본 정보", "guide": "대상·일자·상담 유형"},
    {"title": "호소 문제", "guide": ""},
    {"title": "추후 계획", "guide": "다음 회기 목표와 일정 포함"}
  ]
}
```

> 예전의 단순 문자열 목록 형식(`"sections": ["기본 정보", "호소 문제"]`)도 그대로 읽힙니다.

## 🛡 일지 정확성에 대하여

AI가 메모에 없는 내용을 지어내지 않도록 프롬프트에 강한 안전장치를 넣었습니다.
정보가 부족한 항목은 '추가 확인 필요'로 표기되며, 단정적 표현 대신 관찰 수준으로 기술됩니다.
그래도 **반드시 상담사가 검토·수정 후 사용**하세요.
